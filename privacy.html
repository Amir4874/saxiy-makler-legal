Privacy Policy

Sahiy Makler · Effective date: April 7, 2026 · Last updated: 2026-04-07

1. App Identification

This Privacy Policy explains how Sahiy Makler ("the App", "we", "us") collects, uses, stores, and protects data. Sahiy Makler is operated by Abdulloh Sharifov, an individual operator based in Tashkent, Republic of Uzbekistan ("the Operator"). The App is a real estate video publishing tool that integrates with the TikTok Content Posting API and other social media platforms to publish videos on behalf of authorized users.

2. Data We Collect

2.1 From the user (via Telegram bot input)

  • Real estate listing text (property description, price, location)
  • Property photos uploaded by the user
  • The user's Telegram numeric ID (used solely for access control — only whitelisted IDs may use the bot)
  • Optional caption preferences and publishing choices

2.2 From TikTok

The App uses a pre-authorized TikTok access token, configured by the Operator on the server side. We do not collect any personal information from TikTok users. The token is used exclusively to upload and publish videos via the video.upload and video.publish scopes.

2.3 From other connected platforms

Access tokens for Instagram, YouTube, Facebook, and Odnoklassniki (OK.ru) are stored locally on the Operator's server in environment variables. No profile information, follower data, comments, or analytics are read from any of these platforms.

3. Purpose of Data Collection

We use the data described above only for the following purposes:

  • To translate property listings into Uzbek using a large-language-model API
  • To generate AI voiceover audio (via ElevenLabs text-to-speech)
  • To assemble a vertical short-form video (1080×1920) using FFmpeg
  • To publish the resulting video to the social media accounts the user has explicitly connected and confirmed
  • To enforce access control (only whitelisted Telegram IDs may use the bot)

We do not use the data for advertising, profiling, behavioral analysis, resale, or any purpose other than fulfilling the publishing service the user has requested.

4. TikTok-Specific Data Use

Scopes used: video.upload, video.publish

What the App does with these scopes:

  • Uploads a pre-generated MP4 video file to TikTok via the official Content Posting API endpoint https://open.tiktokapis.com/v2/post/publish/video/init/
  • Publishes the uploaded video as PUBLIC_TO_EVERYONE on the user-authorized TikTok account, only after explicit user confirmation in the Telegram bot
  • Sets standard post settings: comments, duets, and stitch enabled (configurable per post)

What the App does NOT do with these scopes:

  • Read any TikTok user profile information (name, avatar, bio, follower count)
  • List, search, or download videos from any TikTok account
  • Read comments, likes, shares, or any engagement data
  • Fetch analytics, insights, or audience demographics
  • Store any TikTok user identifiers, refresh tokens, or session data beyond the operator-configured access token
  • Share or sell any TikTok-derived data to third parties (we do not derive any such data)

The App is a one-way publishing tool only. It does not read from TikTok.

5. Third-Party Sharing

We do not sell, rent, lease, or share any user data with third parties for marketing, advertising, or any commercial purpose. The only data flow to third parties is the publishing of user-confirmed video content to the social media platforms (TikTok, Instagram, YouTube, Facebook, OK.ru) that the user has explicitly connected.

We use the following processors strictly to deliver the service:

  • Anthropic (Claude API) — for translating listings into Uzbek
  • ElevenLabs — for text-to-speech voiceover generation
  • TikTok, Meta, Google, OK.ru — as publishing destinations

Each of these processors has its own privacy policy that governs their handling of the data we send.

6. Retention Period

  • Temporary media files (audio, video, images created during processing): automatically deleted within 24 hours after publishing, or immediately on processing failure.
  • Property listing text: not persisted after the video is generated and published.
  • Telegram user IDs (whitelist): retained as long as the user is authorized to use the bot; removed within 7 days of revocation.
  • Platform access tokens: stored locally on the Operator's server, encrypted at rest in environment variables; rotated when the Operator regenerates them.
  • Operational logs: retained for up to 30 days for debugging and abuse prevention, then deleted.

  • Access — request a copy of any personal data we hold about you

  • Erasure ("right to be forgotten") — request deletion of your data
  • Portability — request your data in a machine-readable format
  • Restriction — request limitation of processing
  • Objection — object to processing based on legitimate interest
  • Withdraw consent — at any time, without affecting prior lawful processing

If we learn that we have inadvertently collected data from a minor, we will delete it immediately. If you believe a minor has provided us with personal information, please contact abdullohsharifov773@gmail.com.

9. Data Deletion Request

To request deletion of all data associated with you, send an email to abdullohsharifov773@gmail.com with the subject line "Data Deletion Request". Include your Telegram username or ID so we can identify your records. We will:

  1. Confirm receipt within 7 days
  2. Verify the request is genuine (to protect against unauthorized deletion)

  4. Send written confirmation when deletion is complete

10. Security

We protect data using industry-standard practices: encrypted storage of access tokens, HTTPS for all API communication, restricted server access, and a whitelist-based access control model. However, no method of transmission or storage is 100% secure, and we cannot guarantee absolute security.
